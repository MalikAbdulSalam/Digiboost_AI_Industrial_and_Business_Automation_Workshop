📌 1. Automation: Why & What

⏱️ Why automate?

  • ✅ Stop repetitive manual work
  • ✅ Eliminate copy-paste errors
  • ✅ Run tasks 24/7 (while you sleep)
  • ✅ Connect apps without IT skills

🖱️ What is automation?

Let software handle repetitive rules: When X happens → Do Y.

Example: New Gmail arrives → Save attachment to Drive → Post to Slack – all built visually.

⚙️ 2. Core Concepts (drag & drop blocks)

🔔 TRIGGER
“When?”
➡️
🔍 FILTER
“If this?”
➡️
⚡ ACTION
“Do this”
🔔 Trigger nodes: Schedule, Webhook, Email, Form submission, Database change.
🔍 Filtering: IF condition (ex: amount > 100 OR status = active). Use “IF” or “Switch” node.
⚡ Actions: Send email, Create Google Sheets row, Post Slack, WhatsApp message, HTTP request.

📐 3. Workflow & Best Practices (visual)

📌 Workflow = sequence of connected nodes on the canvas
Analogy: recipe steps. Each node = one instruction.

✨ Best practices (no-code style)

  • Name workflows clearly: “WhatsApp_Order_Confirmation” not “test1”
  • Use Edit Fields to clean / combine data
  • Add error handling: connect error output to a “Send Alert” node
  • 🔁 Mapping: click any field → choose {{…}} from previous node – no coding, just selection.
🧩 Mapping example: From “Webhook” node → pick body.customer.email → drag into “Send Email” To field.

🌐 4. API & Webhooks (click, not code)

🔌 Webhook node

1. Drag Webhook trigger → get a unique URL.
2. Paste URL into any app (Stripe, Typeform, Shopify).
3. Event fires → workflow runs automatically.

📡 HTTP Request node

Choose GET/POST, paste endpoint, add headers via UI. Perfect for REST APIs without writing curl.

🧩 5. n8n fundamentals (point & click)

🟢 Nodes – prebuilt blocks: Gmail, Slack, WhatsApp, etc.
🎨 Canvas – infinite workspace to drag & connect nodes.
➕ Adding first node – click “+” → search “Manual Trigger” → add.
⚡ Node Actions – each node has operations (Send, Read, Update).
🔐 Credentials – click “Connect account” → login once → n8n stores encrypted.
📊 Data – JSON flows between nodes; access via expression picker.

🔄 6. Branches (visual IF/ELSE)

➕ Switch node
🔀
✅ Condition 1 → Path A
📌 Condition 2 → Path B
⚡ Default → Path C
💡 Example: Incoming WhatsApp msg → Switch checks keyword “price” → sends price list; keyword “support” → creates Trello ticket.

🛠️ 7. Most useful no‑code nodes

✏️ Edit Fields
Rename, merge, calculate. Example: combine first+last name using UI.
📊 Aggregate
Combine multiple rows → sum, average, count. No SQL needed.
🔗 Webhook (trigger)
Start workflow from external events (payments, forms, etc.)

🐞 8. Execution, Logs & Debugging

FeatureWhat it does (UI based)
📜 Execution HistoryList of every run: status, duration, input/output.
🔴 Error HandlingNodes have separate error output – connect to fallback action.
🧪 Execute NodeTest single node without running whole workflow.
🛑 Wait & RetryAdd “Wait” node before retrying failed steps.
💡 Debug trick: insert “Console” node to inspect data at any point.

👥 9. Collaboration & Team features

📚 Templates – 300+ ready workflows (n8n.com/templates).
👤 User management – roles: Owner, Admin, Member, Viewer.
🔧 Admin – invite users, set SSO, manage instance.
🧑‍🤝‍🧑 Members – see assigned workflows, executions.
🔗 Workflow sharing – share read-only or edit access inside workspace.
🔐 Credential sharing – share credentials with “use only” permission (hide secrets).
⚙️ n8n API – trigger workflows remotely, manage users programmatically.
